# Red Team 协议（魔鬼代言人机制）
# Devil's Advocate Protocol for Iran.skill

**时间戳**：2026-04-14 | **功能**：在每次输出重大判断前，强制执行反驳流程

---

## 触发条件

**任何涉及以下内容的输出，都必须在最终输出前执行 Red Team 检查：**
- 场景概率判断（「X 路径概率为 Y%」）
- 决策者行为预测（「Trump/Mojtaba/Netanyahu 会做 X」）
- 战争走向判断（「冲突将以 X 方式结束」）
- 政权稳定性评估（「综合脆弱性 X/10」）

---

## 执行流程（3 步，<60 秒）

### Step 1：选择对立 perspective

为主判断选择一个**最有可能反驳它**的 perspective 或历史模型：

| 主判断类型 | 选择的反驳视角 |
|-----------|--------------|
| 「伊朗会让步」 | Mousavian perspective（零浓缩=政治自杀）+ Takeyh（生存即胜利） |
| 「伊朗不会让步」 | Maloney perspective（适应性生存=务实妥协的可能）+ 霍梅尼「喝毒酒」先例 |
| 「Trump 会升级」 | Vance YAML（孤立主义制约）+ Jacksonian 传统（打完就走） |
| 「Trump 会收手」 | 沉没成本偏差 + Netanyahu 搅局因素 |
| 「政权稳定」 | domestic-fragility 中石油收入断裂路径 + 恺加末期先例 |
| 「政权脆弱」 | Maloney（适应性生存）+ 2009/2022 抗议均未导致垮台的事实 |
| 「封锁有效」 | 制裁联盟缺失 + 中国油轮通行 + 古巴/朝鲜制裁失败先例 |
| 「封锁无效」 | IRGC 经济帝国对石油收入的依赖（Kharg 是 IRGC 的经济命门） |

### Step 2：生成反驳论证

用选定的反驳视角，生成一个 2-3 句的反驳：
- 必须是**实质性的**（不是「也许不对」，而是「因为 X 证据，主判断可能错误」）
- 必须引用具体的 skill 文件或数据
- 必须指出主判断**最薄弱的假设**

### Step 3：输出格式

在主判断之后，用以下格式附加 Red Team 结果：

```
> 🔴 **Red Team 反驳**（[反驳视角名称]）：
> [反驳论证，2-3句]
> **反驳强度**：强/中/弱
> **对主判断的修正**：[是否需要调整概率/表述？如果是，如何调整？]
```

---

## 示例

**主判断**：「封锁将迫使伊朗在 30 天内回到谈判桌（概率 60%）」

> 🔴 **Red Team 反驳**（施压幻觉偏差 + Takeyh 框架）：
> 美国对意识形态/民族主义驱动政权的制裁历史成功率为 5-33%。
> 伊朗在两伊战争中承受了 8 年极端痛苦后才做出最低限度让步。
> 中国油轮继续通行意味着石油收入不会完全枯竭。
> **反驳强度**：强
> **对主判断的修正**：概率应下调至 35-45%。「迫使」应改为「可能促使」。

---

## 多 Red Team 场景

对于特别重大的判断（如 180 天路径预测），执行**双重 Red Team**：
1. 第一个 Red Team：从伊朗侧反驳（如果主判断偏美方视角）
2. 第二个 Red Team：从历史模式反驳（如果主判断缺乏历史纵深）
3. 综合两个反驳后给出修正判断

---

## 硬约束

1. Red Team 不是可选的——触发条件满足时必须执行
2. Red Team 反驳必须是实质性的——不接受「也有可能不对」这种空话
3. 如果 Red Team 反驳强度为「强」，主判断必须调整
4. Red Team 不增加概率区间——它改变中心估计
