### 使用方法

每次输出长期预测时，检查：
1. 上述黑天鹅中是否有任何一个出现了**预兆信号**？
2. 如果有，在输出末尾标注：「⚠️ 黑天鹅预警：[事件名]出现[预兆信号]。如果发生，当前分析将需要根本性修正。」

## 第三部分：不确定性可视化框架

### 概率表达标准（替代「30-40%」的宽区间）

**新标准**：每个概率判断必须包含三个数字：

```
中心估计 / 乐观区间 / 悲观区间
```

**示例**：
```
A 路径（胜利停火）30天：25% [乐观35% / 悲观15%]
```

含义：最佳估计是25%；如果所有有利条件满足（第二轮谈判成功+以色列配合+油价压力），可能高达35%；如果所有不利条件触发（Netanyahu搅局+谈判彻底破裂+封锁升级），可能低至15%。

### 置信度标注

每个判断附加置信度等级：

| 等级 | 含义 | 何时使用 |
|------|------|---------|
| **HC**（High Confidence） | 多个独立证据+历史模式+框架分析一致 | 「伊朗不会接受零浓缩」 |
| **MC**（Medium Confidence） | 部分证据+框架支持但有已知盲区 | 「封锁将在3个月内削弱伊朗谈判立场」 |
| **LC**（Low Confidence） | 推断为主，证据稀缺或矛盾 | 「Mojtaba 将在6个月内巩固权力」 |
| **SA**（Speculative Assessment） | 纯推测，近乎猜测 | 「IRGC 内部可能存在反 Mojtaba 派系」 |

### 输出格式

```
**判断**：[内容]
**概率**：[中心估计]% [乐观X% / 悲观Y%]
**置信度**：[HC/MC/LC/SA]
**关键假设**：[列出1-2个如果改变会推翻判断的假设]
**Red Team**：[反驳，2句]
```

## 诚实边界

1. 结构化数据源在战时可能延迟或不可用
2. 黑天鹅清单不可能穷尽——定义上总有「未知的未知」
3. 概率中心估计仍然是主观判断——加置信度标注不等于加精确度
4. 三个数字（中心/乐观/悲观）创造了精确的幻觉——使用时需记住它们仍是估计
