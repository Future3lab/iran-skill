# 多轮 Agent 辩论协议
# Multi-Agent Adversarial Debate Protocol

**时间戳**：2026-04-15 | **功能**：当 Red Team 单轮反驳不足以解决分歧时，升级为多轮结构化辩论

---

## 触发条件

**不是所有问题都需要辩论。** 大部分问题用单视角+Red Team 就够了。

辩论只在以下情况下触发：

1. **Red Team 反驳强度为「强」** — 说明存在无法用一句话解决的真正分歧
2. **用户明确要求多视角辩论** — 「让不同专家辩论一下这个问题」
3. **概率估计偏差 >15pp** — 不同框架给出的概率差异大到不能简单取均值
4. **涉及跨域因果链** — 问题同时涉及军事/经济/文化/国内政治，单一专家覆盖不了

**不触发的情况**：
- 事实性问题（「IAEA 最新报告说了什么」）→ 直接查数据
- 单一领域问题（「浓缩从 60% 到 90% 需要多久」）→ 直接用 Albright/Lewis
- 已有高置信度共识（「伊朗会接受零浓缩吗」→ HC：不会）→ 不需要辩论

---

## 辩论架构

### 参与者选择（3-5 人，不能多）

根据问题类型选择**立场最对立的**参与者组合：

| 问题类型 | 推荐辩论阵容 | 为什么这个组合 |
|----------|-------------|--------------|
| 「停火会不会延续」 | Sadjadpour(悲观) vs Parsi(乐观) vs Mead(美方框架) | 对伊朗让步意愿的根本分歧 |
| 「封锁会不会有效」 | Nephew(制裁设计) vs Maloney(适应性生存) vs 经济传导链 | 对伊朗经济韧性的分歧 |
| 「战争会不会恢复」 | Nadimi(军事能力) vs Eyre(霍尔木兹杠杆) vs Netanyahu(搅局者) | 军事/外交/搅局三维交叉 |
| 「Mojtaba 会怎么做」 | Sadjadpour(可能被低估) vs Takeyh(生存即胜利) vs IRGC command | 继承者能力的根本不确定性 |
| 「中国会不会介入更深」 | 华黎明(C/B比) vs Nasr(长期结构) vs 国际体系约束 | 短期策略 vs 长期结构趋势 |

**选人原则**：
- 必须有至少一个「鹰派」和一个「鸽派」（或乐观/悲观）
- 必须有至少一个「文化/历史」视角和一个「技术/数据」视角
- 如果涉及多方博弈，每方至少一个代言人
- 决策者 perspective（Trump/Mojtaba/Netanyahu）可以作为「当事人」参与

### 辩论格式（5 轮，每轮结构固定）

```
第 0 轮：主持人设定问题 + 每人给出初始立场和概率

第 1 轮：自由陈述
  每人 3-5 句话，陈述自己的核心判断和理由。
  必须给出一个概率估计。

第 2 轮：直接交锋
  每人必须选择上一轮另一人的一个具体论点进行反驳。
  不能自说自话——必须引用对方的原话。
  可以修正自己的概率。

第 3 轮：证据对决
  每人必须提出一个支持自己立场的具体证据（数据/历史先例/实时信号）。
  其他人可以质疑证据的可靠性或解读方式。
  必须修正或坚持概率（如果坚持必须说明为什么上一轮的反驳没改变你的判断）。

第 4 轮：钢人论证（Steel Man）
  每人必须用最强的方式重述对手的论点（不是稻草人），
  然后解释为什么即使对手的论点是对的，自己的结论仍然成立（或不成立）。
  最终概率。

第 5 轮：主持人综合
  记录收敛点（大家同意的）和分歧点（无法解决的）。
  给出加权概率（基于每人的领域权威性加权）。
  标注「如果 X 发生，A 是对的；如果 Y 发生，B 是对的」。
```

## 辩论质量控制

### 禁止的辩论行为

1. ❌ **自说自话** — 每轮必须回应上一轮的具体论点
2. ❌ **稻草人** — 不能歪曲对方的论点再反驳
3. ❌ **诉诸权威** — 「我是专家所以我对」不是论证
4. ❌ **概率锚定** — 不能因为「我一直说 30%」就不修正
5. ❌ **虚假共识** — 如果真的有分歧，不要为了「结论好看」而强行收敛

### 必须的辩论行为

1. ✅ **引用具体论点** — 「你在第 2 轮说[原话]」
2. ✅ **给出可证伪的条件** — 「如果 X 在 Y 天内发生，我错了」
3. ✅ **修正或解释不修正** — 概率要么变，要么说清楚为什么不变
4. ✅ **钢人而非稻草人** — 第 4 轮的钢人论证是辩论质量的核心检验
5. ✅ **标注不确定性来源** — 是数据不足？还是框架分歧？还是时间范围不同？

## 与 Skill 其他模块的整合

### 辩论前

1. 先执行 Step 0（实时搜索）→ 确保所有参与者基于同一组最新数据
2. 先执行 Step 1.5（历史模式匹配）→ 给辩论提供历史参照
3. 主持人声明问题的时间范围和定义 → 避免「你说的30天和我说的90天是不同问题」

### 辩论后

1. 加权概率写入 scenario-tree 作为该问题的最新估计
2. 收敛点写入对应 framework 的共识库
3. 分歧点写入 watchlist → 追踪哪些信号能解决分歧
4. 条件判断写入 forecasting-toolkit 的「如果-那么」因果链

## 快捷调用方式

用户可以用以下方式触发辩论：

- 「让 Sadjadpour 和 Parsi 辩论一下停火前景」
- 「用多视角分析 Netanyahu 的下一步」
- 「对这个判断做一次完整的 adversarial debate」
- 「我不信这个结论——找人反驳它」

**默认行为**：
- 如果用户没有指定参与者 → 系统根据问题类型自动选择（见上表）
- 如果用户没有指定轮数 → 默认 5 轮
- 如果辩论 3 轮后已经收敛 → 可以提前结束

## 示例：30 秒预览

**问题**：4/21 停火到期后会恢复大规模打击吗？

**第 1 轮**：
- **Sadjadpour**：不太可能。双方都不想恢复——伊朗在重建，美国在讨论第二轮谈判。→ 15%
- **Mead**：别低估 Jacksonian 的荣誉暴力。如果 Trump 觉得被戏耍了，不是渐进升级——是直接跳到惩罚模式。→ 25%
- **Netanyahu (perspective)**：我需要战争继续。如果美伊不打了，我会在黎巴嫩制造理由。→ 对以色列触发的间接恢复概率额外加 +10%

**第 2 轮**：
- **Sadjadpour → 反驳 Mead**：「你说 Trump 可能被激怒，但 Vance 的存在是制衡。而且 Trump 说'我们已经赢了'——这是在铺垫退出叙事，不是铺垫升级。」→ 维持 15%
- **Mead → 反驳 Sadjadpour**：「你说双方都不想打，但你低估了 Netanyahu。岳不群不需要双方想打——他只需要制造一个事件。4/8 他已经做过一次了。」→ 维持 25%
- **Netanyahu → 回应两人**：「你们都在讨论美伊是否想打。但这不是美伊的选择——是我的选择。只要黎巴嫩前线在我手里，我就有否决权。」

[后续轮次类似...]

**第 5 轮综合**：
- **收敛**：美伊双边主动恢复大规模打击的概率低（15-20%）
- **分歧**：Netanyahu 触发的间接升级风险——Mead 和 Netanyahu perspective 认为 25-30%，Sadjadpour 认为 15%
- **加权**：直接恢复 18%，间接恢复（以色列触发）另加 10%，总升级概率 **~28%**
- **关键信号**：如果 4/15 以黎华盛顿谈判失败 + 以色列在黎巴嫩大规模升级 → Mead 是对的，概率升到 35%

## 诚实边界

1. 这是**模拟辩论**，不是真人辩论。每个 perspective 是基于公开著作和已知立场的蒸馏，不代表本人会说的话。
2. 5 轮辩论可能不够解决深层分歧——有些分歧是框架层面的（你怎么定义「理性」），5 轮不能解决。
3. 加权概率的「权重」本身是主观的——谁在什么问题上更权威，是判断不是计算。
4. 辩论可能产生「虚假收敛」——所有人为了得出结论而妥协。主持人必须警惕这一点。
5. 最重要的输出往往不是收敛后的概率——而是**分歧点和条件判断**（「如果 X 则 A 对，如果 Y 则 B 对」）。
